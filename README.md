# 四位动态数码管显示

本工程演示 FPGA 的动态数码管扫描方法。设计以系统时钟生成计数节拍和位选扫描节拍，循环产生 `0` 到 `9` 的段码，并在四个数码管位之间快速切换。

## 功能与原理

- `i` 在 `0`–`9` 之间循环，驱动 `seg_data` 输出相应七段码。
- `m` 在四个状态间循环，驱动 `sel` 选择当前显示位。
- 利用视觉暂留效果，在四位数码管上形成稳定的动态显示。
- 系统时钟按照 33.33 MHz 配置；源代码中 `CNT_END=166`、`CNT_REFRESH=100` 便于快速观察扫描行为。

```text
clk ─► 分频计数器 ─► div_clk ─► 数字 0~9 计数器 ─► 段码译码 ─► seg_data
  └► 刷新计数器 ─► refre_clk ─► 位选计数器 ────────────────► sel
```

## 顶层接口

| 端口 | 方向 | 说明 |
|---|---|---|
| `clk` | 输入 | 33.33 MHz 系统时钟。 |
| `rst_n` | 输入 | 低电平有效复位。 |
| `en` | 输入 | 原始工程保留端口，当前 RTL 未使用。 |
| `seg_data[7:0]` | 输出 | 七段数码管段选信号。 |
| `sel[2:0]` | 输出 | 数码管位选信号。 |

## 使用方法

在 Vivado Tcl Console 中执行：

```tcl
source create_project.tcl
```

脚本将创建 `xc7z020clg400-1` 工程，设置 `dymc_seg` 为综合顶层，并添加自检仿真。硬件下载前请检查 XDC 中的时钟周期与数码管位选/段选管脚是否匹配当前板卡。

## 仿真与验证

运行行为级仿真后，自检会等待扫描逻辑启动，检查 `seg_data`、`sel` 不含未知态且存在有效位选：

```text
PASS: dynamic seven-segment scan is active
```

该工程已通过 Vivado 综合。由于小数点位未参与译码，综合报告中 `seg_data[7]` 为常量驱动；由于 `en` 没有被 RTL 使用，综合也会报告该输入未连接。这两项均反映原始设计现状。

## 修改建议

- 若要在实际板上使用较慢的数字变化速度，可增大 `CNT_END`。
- 若需要显示任意四位数字，应将常量计数结果改为四个 BCD 输入，并按 `sel` 选择对应的 BCD 位。
- 若 `en` 需要作为显示使能，应在位选或段码输出逻辑中显式使用该信号。
